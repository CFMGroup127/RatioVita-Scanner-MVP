# Gemini conversation capture — New Horizons

## Can Cursor extract the share link automatically?

**No — not reliably.** The public share URL  
[https://gemini.google.com/share/8cba871cd1ca](https://gemini.google.com/share/8cba871cd1ca)  
(title: *Global Culinary Residency Concept*) returns only a **Sign in** shell when fetched programmatically. Google does not expose the full two-way transcript to unauthenticated crawlers.

You must **export the conversation manually** once while signed in.

## 60-second manual capture (recommended)

1. Open the link in Chrome/Safari while signed into Google.
2. Scroll to the **top** of the thread.
3. Select all → Copy (or use Gemini’s **Share / Export** if your account shows it).
4. Paste into `NewHorizons_Blueprint.md` in this folder (replace the `PASTE FULL TRANSCRIPT BELOW` section).
5. Optional PDF: File → Print → Save as PDF → `NewHorizons_Blueprint.pdf`.

## Import into RatioVita

1. Copy `NewHorizons_Blueprint.md` to iCloud Drive or Files on iPad.
2. In RatioVita: **Add receipt → Select files…** → choose the `.md` file.
3. The app routes `.md` / `.txt` through **Manuscript Vault** (not receipt OCR):
   - `HistoricalKnowledgeNode` in **Media Core** (tags: `#NewHorizons`, `#176Yonge`, `#ManuscriptVault`)
   - Library anchor under vault path: `New-Horizons/176-Yonge/Project-Manuscripts-Historical-Vault`

Set **Scan target vault** in Receipts to `New-Horizons/176-Yonge` before import if you want a custom prefix.
